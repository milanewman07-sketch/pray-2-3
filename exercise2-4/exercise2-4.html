Modify your code from the previous exercise to create a checkerboard effect as shown below.

The key to this task actually has nothing to do with loops--it just requires a little maths inside your inner loop. You can think of the problem like this:

  • Every even numbered square in an even numbered row (assuming we're counting from 0 because we're programmers) should be black. Odd numbered squares in even numbered rows should be white.
  • Every odd numbered square in an odd numbered row should be black and every even numbered square in an odd numbered row should be white.

The remainder operator (%, also known as modulo) is very useful for this problem!

When you are done, check your work using check-exercise2-4.html.

For an extension activity, try modifying your solution to use for loops instead of while loops.

Expected output:

Don't modify your loops from the previous exercise.

Inside the inner loop and before you draw the square, add an if-else statement with a placeholder / space for the boolean expression. If the boolean expression is true, set the fill to black. If the boolean expression is false, set the fill to white.

The hardest part is to figure out what the boolean expression should be. As noted above, the remainder operator is very useful here. For all black squares, the x and y coordinates are divisible by the same number. Figure out what that number is and express it using the % (remainder/modulo) operator.
